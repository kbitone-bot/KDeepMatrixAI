## 1. 현재 보유 역량 분석

### 1.1 자산 ①: 합성데이터 생성·검증 플랫폼
**Repo**: `kbitone-bot/synthetic-data-quality-platform`

| 역량 | 상세 | 사업 대응도 |
|---|---|---|
| **합성데이터 생성** | statistical / perturbation / model_based(SDV GaussianCopula) 3가지 모드 | ⭐⭐⭐⭐☆ |
| **품질 검증** | 6차원 평가(분포·상관·프라이버시·제약·결측·ML Utility) | ⭐⭐⭐⭐⭐ |
| **ML 적합성 판정** | TSTR(Train Synthetic, Test Real) 프레임워크 | ⭐⭐⭐⭐⭐ |
| **자동 판정** | PASS/REVIEW/FAIL 3단계 + 치명적 위반 자동 감지 | ⭐⭐⭐⭐☆ |
| **UI/UX** | Streamlit 5탭 + 한국어 네이티브 + Plotly 시각화 | ⭐⭐⭐⭐⭐ |
| **CLI 자동화** | `run_pipeline.py` 배치 실행 | ⭐⭐⭐⭐☆ |
| **리포팅** | JSON(기계판독) + HTML(인터랙티브 대시보드) | ⭐⭐⭐⭐☆ |

**강점**: "생성"이 아닌 **"검증·품질보증(QA)"** 중심의 차별화  
**한계**: 생성된 합성데이터를 **실제 분석모델/AI 모델의 입력으로 연결하는 파이프라인**은 미구현

### 1.2 자산 ②: 빅데이터 분석·가시화 통합 플랫폼 (KDeepMatrixAI)
**Repo**: `kbitone-bot/KDeepMatrixAI`

| 역량 | 상세 | 사업 대응도 |
|---|---|---|
| **모델 자동 인식** | `af_ba_req_*` 패턴 스캔, 메타데이터 수집 | ⭐⭐⭐⭐☆ |
| **RAM 분석** | MTBF/MTTR/고장률/수리율/가용도 + lifelines 분포적합 | ⭐⭐⭐⭐⭐ |
| **시각화** | Plotly 인터랙티브 차트(신뢰도·보전도·고장률·가용도) | ⭐⭐⭐⭐⭐ |
| **API 서버** | FastAPI 기반 REST API (`/api/v1/analyze/*`) | ⭐⭐⭐⭐☆ |
| **웹 UI** | Streamlit 기반 모델 선택·조건 입력·결과 표시 | ⭐⭐⭐⭐⭐ |
| **결과 관리** | `outputs/{analysis_id}/` CSV + HTML 보고서 | ⭐⭐⭐⭐☆ |

**강점**: 기존 분석모델을 **수정하지 않고** 신규 코드만으로 웹 통합 실행  
**한계**: **실시간 모델 Serving**이 아닌 Ad-hoc 배치 분석 구조

## 2. 사업 요구사항 대비 갭(Gap) 분석

사업측에서 문의한 5가지 핵심 질의에 대한 현재 상태:

### 질의 1: "5대 AI 모델을 국방 도메인 특화로 학습된 모델(Binary 파일)로 보유하고 있는지"

| 모델 | 현재 상태 | Binary 보유 | 도메인 특화 학습 | 갭 |
|---|---|---|---|---|
| af_ba_req_001 (RAM) | ✅ 완전 구현 | ❌ 없음 (코드 기반 분석) | ❌ 통계 분포적합만 수행 | **높음** |
| af_ba_req_002 (수명예측) | ⚠️ 구조만 | ❌ 없음 | ❌ 미구현 | **높음** |
| af_ba_req_004 (시뮬레이션) | ⚠️ 구조만 | ❌ 없음 (외부 pickle 필요) | ❌ 미구현 | **높음** |
| af_ba_req_005 (유사부품) | ⚠️ 구조만 | ❌ 없음 (외부 KMeans+TF-IDF 필요) | ❌ 미구현 | **높음** |
| af_ba_req_007 (IMQC) | ⚠️ 구조만 | ❌ 없음 | ❌ 미구현 | **높음** |

**현재 5대 모델은 "Python 분석 스크립트"이지 "학습된 AI 모델 Binary"가 아님**

> 📌 **필요 조치**: 5개 모델 중 1개 이상을 **딥러닝/머신러닝 모델로 재구현**하여 `.pkl`/`.joblib`/`.onnx` Binary를 생성해야 함.  
> 특히 af_ba_req_004(시뮬레이션)은 이미 XGBoost/SVR/RandomForest 기반이므로 **가장 쉽게 Binary화 가능**.

---

### 질의 2: "국방지능형플랫폼 Serving 영역에 배포하여 Inference까지 수행 가능한지"

| 요구 항목 | 현재 상태 | 필요 기술/플랫폼 |
|---|---|---|
| 모델 패키징 (Binary) | ❌ 없음 | pickle, joblib, ONNX |
| 모델 레지스트리 | ❌ 없음 | MLflow Model Registry, DVC |
| REST API Serving | ⚠️ FastAPI 있으나 분석용 | BentoML, Triton, TorchServe |
| 컨테이너화 | ❌ 없음 | Docker, Kubernetes |
| A/B 테스트 | ❌ 없음 | Istio, FlagSmith |
| 모니터링 | ❌ 없음 | Prometheus, Grafana |
| 버전 관리 | ❌ 없음 | MLflow, DVC |

**현재 KDeepMatrixAI의 FastAPI는 "분석 실행 API"이지 "모델 Inference API"가 아님**

> 📌 **필요 조치**: 
> 1. 학습된 모델 Binary를 산출
> 2. BentoML 또는 순수 FastAPI로 **모델 Serving 서비스**를 별도 구축
> 3. Docker 이미지화 + Kubernetes Manifest 작성
> 4. 국방지능형플랫폼의 표준 API 스펙(요청/응답 포맷)을 준수하도록 어댑터 개발

---

### 질의 3: "시뮬레이션 데이터 보유 여부"

| 모델 | 시뮬레이션 데이터 존재 여부 | 상태 |
|---|---|---|
| af_ba_req_001 | ✅ `가용도분석자료.xlsb` (327,699행) | 실데이터 |
| af_ba_req_002 | ✅ `정밀측정폐품현황.xlsb` (31,206행) | 실데이터 |
| af_ba_req_004 | ❌ `.py` 코드만 존재, 학습된 pickle/CSV 없음 | **시뮬레이션 데이터 부재** |
| af_ba_req_005 | ✅ `정밀측정품목현황.xlsb` (9.8MB) | 실데이터 |
| af_ba_req_007 | ✅ `21-25년 계획수립현황.xlsx` (465,372행) | 실데이터 |

**af_ba_req_004는 시뮬레이션 데이터가 없어 현재 실행 불가**

> 📌 **필요 조치**: 
> 1. af_ba_req_004의 학습된 pickle 모델과 inference 기준 CSV를 확보하거나,
> 2. 코드를 기반으로 **새로운 시뮬레이션 데이터를 생성**하여 학습 → Binary화

---

### 질의 4: "실 구축 후 해당 데이터를 이용한 검증 시나리오 사례와 시연"

현재 보유한 **검증** 역량:
- ✅ 합성데이터 품질 검증 (6차원, TSTR)
- ✅ RAM 분석 결과 시각화 검증
- ❌ **합성데이터 → AI 모델 학습 → 실데이터 Inference 성능 비교** 시나리오 미구현
- ❌ **시뮬레이션 데이터 vs 실데이터 vs 합성데이터** 3자 비교 시나리오 미구현

> 📌 **필요 조치**: 
> 1. "실데이터 → 합성데이터 생성 → 동일 모델로 각각 학습 → 동일 테스트셋으로 성능 비교" 파이프라인 구축
> 2. af_ba_req_004를 시연용 핵심 모델로 선정하여 **실데이터 예측 vs 시뮬레이션 예측 vs 합성데이터 기반 예측**을 한 화면에서 비교하는 데모 개발

### 3.2 신규 개발 필요 모듈

| # | 모듈명 | 기능 | 기술 스택 | 우선순위 |

## 5. 시연 시나리오 제안 (데모 구성)

### 시나리오 A: "합성데이터로 학습한 RAM 모델의 신뢰성 검증"
```
[Step 1] 실데이터(가용도분석자료.xlsb) 업로드
    ↓
[Step 2] 합성데이터 생성 (perturbation 모드, 327,699행 동일 규모)
    ↓
[Step 3] 품질 검증 실행 → PASS 판정 확인
    ↓
[Step 4] 실데이터로 RAM 분석 실행 → MTBF=330.66, Availability=0.9428
    ↓
[Step 5] 합성데이터로 동일 RAM 분석 실행 → 지표 비교
    ↓
[Step 6] 두 결과를 나란히 시각화 + 차이율 리포트 출력
```

### 시나리오 B: "시뮬레이션 역추정 모델의 Serving 데모"
```
[Step 1] 시험소별 작업량 예측 모델(af_ba_req_004)을 BentoML로 Serving
    ↓
[Step 2] 입력: 난이도, 기술등급, 효율%, 정비주기, 소모인시
    ↓
[Step 3] 예측: 주간 작업량 + Bootstrap 90% CI + Conformal Prediction PI
    ↓
[Step 4] 역추정: 목표 작업량 달성을 위한 최적 입력 변수 자동 산출
    ↓
[Step 5] 결과를 Plotly 대시보드로 실시간 시각화
```

### 시나리오 C: "3자 데이터 A/B 테스트"
```
[Step 1] 동일 모델(예: XGBRegressor)을 3가지 데이터셋으로 각각 학습
    ├─ Model-Real: 실데이터 학습
    ├─ Model-Syn: 합성데이터 학습
    └─ Model-Sim: 시뮬레이션 데이터 학습
    ↓
[Step 2] 동일 테스트셋(실데이터)로 3 모델 Inference
    ↓
[Step 3] 성능 비교 (RMSE, MAE, R²) + 통계적 유의성 검정
    ↓
[Step 4] "합성데이터가 실데이터의 95% 성능을 달성함" 등의 정량적 근거 도출
```

## 6. 결론 및 제언

### 6.1 현재 위치
- **합성데이터 플랫폼**: 생성·검증은 세계적으로도 경쟁력 있는 수준
- **분석 통합 플랫폼**: af_ba_req_001은 완전 구현, 나머지 4개는 구조만 등록
- **연결고리**: 두 플랫폼이 **물리적으로 분리**되어 있어 "합성데이터 → 분석 → 검증" End-to-End 흐름이 자동화되지 않음

### 6.2 최우선 과제 (즉시 실행 필요)
1. **af_ba_req_004 완전 구현 + Binary화**: 시뮬레이션 모델은 이미 ML 기반이라 가장 빠르게 Serving 가능
2. **Model Serving Layer 추가**: BentoML 기반으로 1개 모델이라도 실시간 Inference 데모 확보
3. **3자 비교 시나리오 파이프라인**: "실데이터 vs 합성데이터 vs 시뮬레이션"을 한 화면에서 보여주는 데모는 **제안 발표의 핵심 임팩트**

### 6.3 사업 제안 전략
- **차별화 포인트**: "단순 합성데이터 생성"이 아닌 **"합성데이터 기반 국방 AI 모델 학습·검증·배포 전 주기 플랫폼"**
- **실증 근거**: TSTR + 3자 A/B 테스트로 "합성데이터가 실전 투입 가능하다"는 정량적 근거 확보
- **확장성**: Adapter 패턴으로 향후 새로운 국방 분석모델을 무한히 추가 가능함을 강조
